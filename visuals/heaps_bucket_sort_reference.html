What Is a Heap?

A heap is a complete binary tree that satisfies the heap property. It is stored as an array — no pointers needed.

Complete binary tree

Every level is fully filled except possibly the last level, which is filled from the left. This is what makes array storage work — there are no gaps.

Heap property — Max Heap

Every parent node is greater than or equal to both its children. The largest element is always at the root (index 0 in Python).

Heap property — Min Heap

Every parent node is less than or equal to both its children. The smallest element is always at the root. Python's heapq is a min heap.

NOT a BST

In a BST, left child < parent < right child — giving you sorted order. In a heap, left and right children have no ordering relationship with each other. Only parent vs children is guaranteed.

Array Representation

A complete binary tree maps perfectly onto an array. No pointers. The parent-child relationships are computed from the index.

Heap Operations & Complexity

OperationComplexityHow it works
insert(val) O(log n) Add to end of array (last position in tree), then bubble up — swap with parent while heap property is violated. At most log n swaps (tree height).
extract_max() / extract_min() O(log n) Remove root. Move last element to root. Bubble down (heapify down) — swap with larger child (max heap) until heap property restored. At most log n swaps.
peek() / get_max() / get_min() O(1) Root is always the max (or min). Just return index 0.
heapify(array) O(n) Build a heap from an unsorted array. Process from middle to root, bubble down each. Surprisingly O(n) not O(n log n) — most nodes are near the bottom and do little work.
heappush + heappop × k O(n log k) For Top K Frequent: push n items O(log k) each. This is why bucket sort beats it for Top K — bucket sort is O(n) flat.

Bubble Up vs Bubble Down

Bubble Up (after insert)

After inserting at the end, compare with parent. If violates heap property, swap. Repeat until fixed or reach root.

# insert 25 into max heap [21,17,16,14,9,12,13]
# 1. append: [..., 25] — at index 7
# 2. parent of 7 = (7-1)//2 = 3 → value 14
# 3. 25 > 14 → swap
# 4. now at index 3, parent = 1 → value 17
# 5. 25 > 17 → swap
# 6. now at index 1, parent = 0 → value 21
# 7. 25 > 21 → swap → 25 is new root ✓

Bubble Down (after extract)

Remove root. Put last element at root. Compare with both children, swap with the larger (max heap). Repeat until fixed or reach leaf.

# extract max (21) from [21,17,16,14,9,12,13]
# 1. remove 21, put last (13) at root
# 2. [13, 17, 16, 14, 9, 12]
# 3. children of 0: index 1 (17), index 2 (16)
# 4. 13 < 17 → swap with larger child
# 5. [17, 13, 16, 14, 9, 12]
# 6. children of 1: index 3 (14), index 4 (9)
# 7. 13 < 14 → swap
# 8. [17, 14, 16, 13, 9, 12] ✓

How Bucket Sort Works for Top K Frequent

Key insight: the maximum frequency any element can have is n (if all elements are the same). So we create n + 1 buckets, where the index = frequency. Each bucket is a list of elements with that frequency.

Input: [1, 1, 2, 1, 2, 55] — k = 2

Step 1 — Count frequencies (hashmap):

2 → 2

← iterate backwards from index 6 → pick top k elements

Step 3 — Read from high frequency to low, collect k elements:

freq 6 → empty | freq 5 → empty | freq 4 → empty | freq 3 → [1] ✓ | freq 2 → [2] ✓ | k=2 reached → return [1, 2]

💡 Why O(n)? Counting frequencies is O(n). Building the buckets array is O(n). Reading back is O(n). No sorting — you never compare elements against each other. The frequency index does the ordering implicitly.

Bucket Sort — The Code

def topKFrequent(nums, k):
    # Step 1: count frequencies
    count = {}
    for num in nums:
        count[num] = count.get(num, 0) + 1

    # Step 2: create n+1 buckets (index = frequency)
    # bucket[i] = list of elements with frequency i
    freq = [[] for _ in range(len(nums) + 1)]
    for num, cnt in count.items():
        freq[cnt].append(num)

    # Step 3: read from high frequency to low
    result = []
    for i in range(len(freq) - 1, 0, -1):
        for num in freq[i]:
            result.append(num)
            if len(result) == k:
                return result

Two Approaches to Top K Frequent

ApproachTimeSpaceWhen to use
Bucket Sort O(n) O(n) When you need the best possible time complexity. Frequency range is bounded by n so buckets don't blow up.
Min Heap of size k O(n log k) O(k) When k is much smaller than n and memory matters more than speed. Only keeps k elements in the heap at once.
💡 For the interview: if they ask "what's the optimal solution?", it's bucket sort at O(n). If they ask "can you do it with a heap?", explain the O(n log k) approach. Knowing both shows depth.

Bucket Sort Approach

def topKFrequent(nums, k):
    count = {}
    for num in nums:
        count[num] = count.get(num, 0) + 1

    freq = [[] for _ in range(len(nums) + 1)]
    for num, cnt in count.items():
        freq[cnt].append(num)

    result = []
    for i in range(len(freq) - 1, 0, -1):
        for num in freq[i]:
            result.append(num)
            if len(result) == k:
                return result

# Time: O(n) — count O(n) + build buckets O(n) + read O(n)
# Space: O(n) — count dict + freq array both O(n)

Heap Approach (O(n log k))

import heapq

def topKFrequent(nums, k):
    count = {}
    for num in nums:
        count[num] = count.get(num, 0) + 1

    # heapq.nlargest: O(n log k)
    # key=lambda: sort by frequency value, not the number itself
    return heapq.nlargest(k, count.keys(), key=lambda x: count[x])

# OR manually with a min heap of size k:
def topKFrequent_heap(nums, k):
    count = {}
    for num in nums:
        count[num] = count.get(num, 0) + 1

    heap = []
    for num, cnt in count.items():
        heapq.heappush(heap, (cnt, num))   # push (freq, num)
        if len(heap) > k:
            heapq.heappop(heap)             # remove least frequent

    return [num for cnt, num in heap]

# Time: O(n log k) — n pushes, each O(log k) since heap stays size k
# Space: O(k) — heap only holds k elements at a time
⚠ The lambda in key=lambda x: count[x] — this tells heapq to rank by frequency (count[x]), not by the value of the number itself. Without this, it would sort by the numbers (1, 2, 55) not by how often they appear (3, 2, 1).

Python heapq — Key Facts

Min heap only

Python's heapq is always a min heap. The smallest element is always at index 0. To simulate a max heap, negate your values: push -val, pop and negate the result.

heapq operates on a list

You don't create a Heap object. You pass a regular Python list to heapq functions. The list is modified in-place to maintain the heap invariant.

Tuple ordering

Push tuples (priority, value) to control ordering. heapq compares tuples element by element — first by priority, then by value if tied.

heapq Cheat Sheet

import heapq

heap = []

# Push — O(log n)
heapq.heappush(heap, 5)
heapq.heappush(heap, 1)
heapq.heappush(heap, 3)
# heap = [1, 5, 3] — 1 is at root (min)

# Pop min — O(log n)
val = heapq.heappop(heap)   # returns 1

# Peek min — O(1)
val = heap[0]               # don't pop, just look

# Build heap from list — O(n)
nums = [3, 1, 4, 1, 5]
heapq.heapify(nums)          # modifies in place

# Get k largest — O(n log k)
top3 = heapq.nlargest(3, nums)

# Get k smallest — O(n log k)
bot3 = heapq.nsmallest(3, nums)

# Max heap — negate values
heapq.heappush(heap, -21)   # push negated
max_val = -heapq.heappop(heap)  # negate on pop

# Tuple push — sort by first element
heapq.heappush(heap, (3, "apple"))
heapq.heappush(heap, (1, "banana"))
heapq.heappop(heap)  # returns (1, "banana")
